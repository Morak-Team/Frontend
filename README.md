## 🍃 Branch Convention

- **`main`**
    - 배포용 브랜치
- **`develop`**
    - 개발 통합용 브랜치
- **`feature/#이슈번호/개발 내용`**
    - 기능 단위 브랜치

## 👾 Commit Convention

**📦 초기 설정**

- `Init` – 프로젝트 초기 세팅에만 사용
    
**✨ 기능 관련**

- `Feat` – 새로운 기능 추가
- `Fix` – 버그 수정
- `!HOTFIX` – 치명적인 긴급 버그 수정

**🎨 스타일 및 디자인**

- `Style` – 코드 로직 변경 없이 스타일만 수정
- `Design` – UI/UX 중심의 사용자 화면 디자인 변경

**🔁 코드 개선**

- `Refactor` – 기능 변경 없이 코드 리팩토링
- `Comment` – 주석 추가 또는 수정

**🧹 파일 및 구조**

- `Rename` – 파일 또는 폴더 이름 변경 또는 이동
- `Remove` – 파일 삭제만 수행

**🧪 테스트 및 기타**

- `Test` – 테스트 코드 추가 또는 리팩토링
- `Docs` – 문서 추가, 수정, 삭제
- `Chore` – 기타 자잘한 변경 (코드 변경 없이 설정 등)
